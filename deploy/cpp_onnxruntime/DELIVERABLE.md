# StarryOS / QEMU ACT CPU 推理 — 实现与验证

## 交付内容

| 交付物 | 说明 |
| --- | --- |
| RISC-V 推理程序 | C++ + ONNX Runtime，约 172 KB |
| 量化 ONNX 模型 | 静态 QDQ，动作头 FP16，约 50 MB |
| FP32 参考模型 | 约 194 MB（仅用于对比评测，未随仓库精简版提交） |
| 归一化参数 | 图像与状态/动作分位数配置 |
| 评测清单 | 666 帧闭环数据 |

## 推理流程

1. 解码 RGB，320×240 → 224×224，ImageNet 归一化。  
2. 状态 `[left_vel, right_vel]` 按分位数映射到 `[-1, 1]`。  
3. ONNX 输入：`image [1,3,224,224]`、`state [1,2]`、`latent [1,32]`（固定均值）。  
4. 输出动作块反归一化，取**第一步** `[左轮速, 右轮速, 夹爪]`。  
5. 按左右轮速差与阈值判定直行 / 左转 / 右转。

## 部署与运行方式（简述）

- 交叉编译得到 RISC-V 可执行文件，与 ONNX Runtime 库、量化模型、参数及评测图像一并装入 StarryOS 根文件系统。  
- 使用 **QEMU + StarryOS** 启动（内存 1G、单核），在系统内运行推理程序。  
- 内核侧对 `/proc` 做了小补丁，以便读取 **VmRSS / VmHWM**（见补丁说明章节）。

## 单帧验证（StarryOS / QEMU）

初态 `[0, 0]` 示例输出：

```text
first_step: left_vel=0.000542187 right_vel=3.93152e-05
diff=0.000502872 decision=straight
```

另有一帧左转样例：预测方向与真值一致，详见「单帧推理结果样例」章节。

## 闭环评测（666 帧）

- 每个 episode 首帧用真值初始状态，之后用**上一帧预测轮速**作为当前状态（闭环）。  
- 统计轮速差 MAE/RMSE；转向准确率时**忽略真值为直行的帧**。

**选定量化模型结果：**

```text
样本数：666
diff MAE：0.019771
|误差|≤0.01：89.79%
忽略“预测为直行”后的转向符号准确率：92.31%
反向预测：1 次
```

模型偏保守（不少真值转向被预测为直行），但一旦给出转向，左右关系与真值高度一致，满足赛题对决策方向的要求。

## 量化方案对比（eps=0.005）

下表用中文概括几种方案；完整评测表里保留英文代号（`static`、`conv_matmul`、`head_fp16` 等），含义见第六章「表中英文名称怎么读」。

| 方案 | diff MAE | diff≤0.01 | 忽略预测直行后符号准确率 |
| --- | --- | --- | --- |
| FP32 | 0.0221 | 0.893 | 0.818 |
| **选定：静态 Conv/MatMul INT8 + 动作头 FP16** | **0.0198** | **0.898** | **0.923** |
| 动态 attention/FFN INT8 + 动作头 FP16 | 0.0213 | 0.841 | 0.576 |

选定理由：体积小、轮速差误差更好、给出转向时符号准确率高、反向预测极少。

## 资源与耗时（StarryOS / QEMU）

| 阶段 | 耗时 | 常驻内存 VmRSS |
| --- | --- | --- |
| 加载参数 | 10.5 ms | 2.7 MB |

| 创建 Session | 8015 ms | 75.8 MB |
| 图像预处理 | 32.2 ms | 76.4 MB |

进程峰值 **VmHWM ≈ 76.8 MB**，虚拟地址约 90 MB。

## 稳定性

- 666 帧闭环全程复用同一推理会话，无崩溃。  
- 单帧与批量运行中内存读数稳定。
